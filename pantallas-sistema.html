Entregable 3 — Diseño de interfaces

Pantallas del sistema propuesto

Wireframes y descripción de funcionamiento de las 5 pantallas nuevas que se integran al flujo existente de catálogo, carrito y pago de IPR.

El flujo de compra actual (catálogo → carrito → pago) ya existe en la página web de IPR y no se modifica. Las pantallas descritas a continuación son las 5 nuevas interfaces que se activan a partir del momento en que el cliente confirma el pago.

Pantalla 01 · Cliente

Confirmación y estado del pedido en tiempo real

👨‍🔧 Actor: Cliente técnico / distribuidor

Se muestra el seguimiento en tiempo real del pedido y la bodega asignada, reemplazando el mensaje genérico de espera.

↗ HU-01 · HU-02 · HU-04
Pantalla 02 · Operaciones internas

Panel de órdenes de despacho — Bodega

📦 Actor: Personal de bodega

La orden llega automáticamente a bodega al confirmar pago, sin papel ni gestión manual del asesor.

↗ HU-03 · HU-05
Pantalla 03 · Transportador / Asesor

Registro y gestión de novedades post-despacho

🚚 Actor: Transportador → 💼 Asesor comercial

El transportador registra novedades y el asesor recibe alertas para resolver excepciones desde el sistema.

↗ HU-06
Pantalla 04 · Asesor comercial

Bandeja de pedidos especiales para aprobación

💼 Actor: Asesor comercial

Solo llegan casos especiales (crédito/volumen/precio), dejando pedidos estándar automatizados.

↗ HU-08
Pantalla 05 · Coordinador logístico

Dashboard operacional y calificación post-entrega

📊 Actor: Coordinador + Cliente

Incluye calificación del cliente y vista KPI operativa para seguimiento y mejora continua.
